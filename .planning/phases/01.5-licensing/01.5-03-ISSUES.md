### UAT-001: Subscribe endpoint fails with depositType mismatch

**Discovered:** 2026-01-13
**Resolved:** 2026-01-13
**Commit:** `6e1129f`
**Phase/Plan:** 01.5-03
**Severity:** Blocker
**Feature:** POST /api/crypto/subscribe endpoint
**Description:** When calling the subscribe endpoint, the NEAR Intents API returns "ANY_INPUT only supports depositType INTENTS" error.
**Expected:** Endpoint should create a subscription with deposit address and return intentId, depositAddress, monthlyAmount
**Actual:** Returns 500 error because the code uses `depositType: 'ORIGIN_CHAIN'` but ANY_INPUT swap type requires `depositType: 'INTENTS'`
**Resolution:** Changed `depositType: QuoteRequest.depositType.ORIGIN_CHAIN` to `depositType: QuoteRequest.depositType.INTENTS` in `workers/license-api/src/services/near-intents.ts` line 70.
